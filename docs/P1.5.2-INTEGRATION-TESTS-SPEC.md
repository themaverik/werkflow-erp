## Testing Approach

**Setup Pattern:**
```java
@WebMvcTest(/* controller */)
@Import({...Services, ...Repositories...})
@MockBean({TenantContext.class, UserContext.class, ...})
class WorkflowIntegrationTest {

    @Autowired
    private MockMvc mockMvc;

    @Autowired
    private SomeService service;

    @BeforeEach
    void setUp() {
        // Mock TenantContext to return specific tenant
        when(tenantContext.getTenantId()).thenReturn("ACME");
    }

    @Test
    void testWorkflow() throws Exception {
        // 1. Create via HTTP
        mockMvc.perform(post("/api/v1/...").contentType(APPLICATION_JSON).content(...))
            .andExpect(status().isCreated())
            .andDo(result -> id = extractId(result.getResponse().getContentAsString()));

        // 2. Verify via service
        SomeResponse response = service.getSomething(id);
        assertNotNull(response.getCreatedByDisplayName());

        // 3. Update via service
        service.updateStatus(id, NEW_STATUS);

        // 4. Verify via HTTP
        mockMvc.perform(get("/api/v1/.../{id}", id))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.status").value(NEW_STATUS.toString()));
    }
}
```

**Key Patterns:**
- Use `@WebMvcTest` + manual `@Import` of required beans (fast startup)
- Mock TenantContext and UserContext to control tenant/user
- Mix HTTP calls (MockMvc) and direct service calls
- Verify both HTTP response AND service state
- Check audit fields (createdByDisplayName, updatedByDisplayName)
- Test error cases: 403, 404, 400

**Assertions:**
- Status codes (200, 201, 400, 403, 404)
- JSON paths for response fields
- Service state after operations
- Audit field population
- Tenant isolation enforcement
