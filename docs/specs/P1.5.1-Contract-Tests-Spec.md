## EmployeeService Contract Tests

**Location**: `services/business/src/test/java/com/werkflow/business/hr/service/EmployeeServiceTest.java`

**Key Contracts to Test:**

1. **Duplicate Email Validation**
   - Create employee with email that already exists → throws `IllegalArgumentException`
   - Message contains "Email already exists"

2. **Duplicate Keycloak User ID Validation**
   - Create employee with keycloakUserId already linked → throws `IllegalArgumentException`
   - Message contains "Keycloak user already linked"

3. **Link Keycloak User - Idempotent**
   - Link same keycloakUserId twice → both succeed (no error)
   - Contract: linking is idempotent

4. **Link Different Keycloak User - Conflict**
   - Employee already linked to keycloakUserId A
   - Try to link to keycloakUserId B → throws `DataIntegrityViolationException`
   - Message contains "already linked to a different Keycloak user"

5. **Department Head Uniqueness**
   - Dept head (doaLevel=2) already exists for department code "SALES"
   - Try to create new dept head for same dept → throws `IllegalStateException`
   - Message contains "department head already exists"

6. **Tenant Isolation**
   - Employee exists but belongs to different tenant
   - Try to access via current tenant context → throws `EntityNotFoundException`
   - Contract: no cross-tenant data leakage

## BudgetCheckService Contract Tests

**Location**: `services/business/src/test/java/com/werkflow/business/finance/service/BudgetCheckServiceTest.java`

**Key Contracts to Test:**

1. **No Budget Plan Found**
   - No budget plan for department in fiscal year
   - Response: `available=false`, reason contains "No budget plan found"
   - `availableAmount=0`, `allocatedAmount=0`

2. **Sufficient Budget**
   - Allocated: 100,000, Spent: 20,000, Requested: 50,000
   - Available: 80,000 (allocated - spent)
   - Response: `available=true`, availableAmount=80,000

3. **Insufficient Budget**
   - Allocated: 50,000, Spent: 40,000, Available: 10,000
   - Requested: 15,000
   - Response: `available=false`, reason contains both amounts

4. **Fiscal Year Default**
   - No fiscal year in request → defaults to current year
   - Response contains current year

5. **Exact Match**
   - Allocated: 100,000, Spent: 70,000, Available: 30,000
   - Requested: 30,000
   - Response: `available=true`

6. **Zero Allocated**
   - Allocated: 0, Spent: 0, Requested: 1,000
   - Response: `available=false`

## Testing Approach

**Use Mockito mocks for all dependencies:**
- Repositories → use `when().thenReturn()`
- TenantContext → return fixed tenant ID
- Validators → throw exceptions as needed

**No entity builders needed:**
- Create simple objects with all fields set directly
- Focus on service contract, not entity construction

**Pattern:**
```java
@Mock
private SomeRepository repo;

@Mock
private TenantContext tenantContext;

private ServiceUnderTest service;

@BeforeEach
void setUp() {
    MockitoAnnotations.openMocks(this);
    service = new ServiceUnderTest(repo, tenantContext);
    when(tenantContext.getTenantId()).thenReturn("ACME");
}

@Test
void testContractBehavior() {
    // Arrange: mock repository calls
    when(repo.findById(1L)).thenReturn(Optional.of(mockObject));

    // Act: call service
    ResultType result = service.operation(input);

    // Assert: verify contract
    assertNotNull(result);
    assertEquals(expected, result.field);
}
```
