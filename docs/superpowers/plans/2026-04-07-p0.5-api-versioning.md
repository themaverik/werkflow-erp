# P0.5 API Versioning Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Change the application context path from `/api` to `/api/v1` and update all documentation to reflect the new endpoint paths.

**Architecture:** Spring Boot's configuration-only approach: modify `server.servlet.context-path` in application.yml, update Swagger OAuth2 redirect URL, and update all documentation files with new endpoint URLs. No controller code changes needed — Spring handles routing transparently.

**Tech Stack:** Spring Boot configuration, YAML, Markdown documentation

---

## File Structure

| File | Action | Responsibility |
|------|--------|-----------------|
| `services/business/src/main/resources/application.yml` | Modify | Change context-path and oauth2-redirect-url |
| `docs/WERKFLOW_INTEGRATION.md` | Modify | Update hardcoded endpoint URLs and examples |
| `docs/Independence-Checklist.md` | Modify | Update curl command examples |
| `Implementation-Summary.md` | Modify | Update example URLs in quick-start |

## Tasks

### Task 1: Update application.yml context-path

**Files:**
- Modify: `services/business/src/main/resources/application.yml` (lines 60-92)

- [ ] **Step 1: Read current application.yml to see the structure**

Run:
```bash
cat services/business/src/main/resources/application.yml | grep -A 30 "^server:"
```

Expected: Output shows current `context-path: /api` and swagger config.

- [ ] **Step 2: Update context-path in application.yml**

Find the line:
```yaml
server:
  port: ${SERVER_PORT:8084}
  servlet:
    context-path: /api
```

Replace with:
```yaml
server:
  port: ${SERVER_PORT:8084}
  servlet:
    context-path: /api/v1
```

- [ ] **Step 3: Update oauth2-redirect-url in Swagger config**

Find the line:
```yaml
    oauth2-redirect-url: http://localhost:8084/swagger-ui/oauth2-redirect.html
```

Replace with:
```yaml
    oauth2-redirect-url: http://localhost:8084/api/v1/swagger-ui/oauth2-redirect.html
```

- [ ] **Step 4: Verify the changes**

Run:
```bash
cat services/business/src/main/resources/application.yml | grep -A 5 "context-path\|oauth2-redirect-url"
```

Expected: Shows `context-path: /api/v1` and redirect URL with `/api/v1/`.

- [ ] **Step 5: Compile to verify no YAML syntax errors**

Run:
```bash
cd services/business
mvn clean compile -q
```

Expected: BUILD SUCCESS, no errors.

- [ ] **Step 6: Commit**

```bash
git add services/business/src/main/resources/application.yml
git commit -m "feat(P0.5.1): change context-path from /api to /api/v1"
```
