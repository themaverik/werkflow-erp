**Goal:** Implement a PATCH endpoint to link Keycloak user IDs to employees, called by Admin Service after user provisioning.

**Architecture:** Single idempotent endpoint that updates the `keycloakUserId` field on Employee entity. Uses Spring Security JWT for authentication and TenantContext for multi-tenant isolation. Conflict detection prevents relinking to different Keycloak users. Exception handling via GlobalExceptionHandler (P1.1).

**Tech Stack:** Spring Boot REST, JPA/Hibernate, TenantContext, Spring Security, GlobalExceptionHandler, JUnit 5, Mockito

Add this method to `EmployeeService`:

```java
/**
 * Link a Keycloak user ID to an employee.
 * Called by Admin Service after user provisioning in Keycloak.
 * Idempotent: calling with same keycloakUserId multiple times is safe.
 *
 * @param employeeId Employee ID
 * @param keycloakUserId Keycloak user UUID from Keycloak
 * @return Updated EmployeeResponse with keycloakUserId
 * @throws EntityNotFoundException if employee not found or tenant mismatch
 * @throws DataIntegrityViolationException if already linked to different keycloak user
 */
public EmployeeResponse linkKeycloakUser(Long employeeId, String keycloakUserId) {
    // 1. Fetch employee
    Employee employee = employeeRepository.findById(employeeId)
        .orElseThrow(() -> new EntityNotFoundException("Employee with ID " + employeeId + " not found"));

    // 2. Validate tenant ownership (multi-tenant isolation)
    String currentTenant = TenantContext.getTenantId();
    if (!employee.getTenantId().equals(currentTenant)) {
        throw new EntityNotFoundException("Employee not found"); // Don't leak tenant info
    }

    // 3. Conflict detection: if already linked to different keycloak user
    if (employee.getKeycloakUserId() != null &&
        !employee.getKeycloakUserId().equals(keycloakUserId)) {
        throw new DataIntegrityViolationException(
            "Employee already linked to different Keycloak user. " +
            "Current: " + employee.getKeycloakUserId() + ", attempted: " + keycloakUserId
        );
    }

    // 4. Set keycloak user ID (no-op if already set to same value)
    employee.setKeycloakUserId(keycloakUserId);

    // 5. Save (atomic update)
    employee = employeeRepository.save(employee);

    // 6. Return updated response
    return EmployeeResponse.fromEntity(employee);
}
```

        EmployeeResponse response = EmployeeResponse.builder()
            .id(1L)
            .firstName("John")
            .lastName("Doe")
            .email("john@example.com")
            .keycloakUserId("keycloak-uuid-123")

            .andExpect(jsonPath("$.keycloakUserId").value("keycloak-uuid-123"));

    @Test
    void testLinkKeycloakUserNotFound() throws Exception {

        // Execute & Verify
        mockMvc.perform(patch("/employees/999/keycloak-link")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(request)))
            .andExpect(status().isNotFound());

    void testLinkKeycloakUserConflict() throws Exception {
